관련 이슈 #158

문제 상황

운영서버내 외부 API(OpenAI, Claude)와의 동기 통신으로 인한 병목으로 p99 응답 시간이 최대 13초까지 증가하여 사용자 경험이 크게 저하되었습니다. 태그 생성, 레퍼런스 생성을 순차 처리하여 단일 요청에도 평균 3~5초가 소요되었습니다.

p99 기준 13초 소요되는 API 포착

해결 과정

@TransactionalEventListener와 @Async를 활용하여 AI API 호출(태그 생성, 레퍼런스 생성)을 비동기로 처리하도록 개선했습니다. TIL 생성 후 즉시 응답을 반환하고, 부가 기능들은 백그라운드에서 처리하여 응답 시간을 단축했습니다.

관련 이슈 #167 기술 블로그

문제 상황

외부 API(OpenAI, Claude)의 갑작스러운 장애로 인해 TIL 태그 및 레퍼런스 데이터가 유실되는 치명적인 문제가 발생하여 서비스 신뢰도에 영향을 주었습니다. 일회성 장애로 인한 데이터 무결성 훼손이 주요 이슈였습니다.

정상적인 호출(태그,레퍼런스있음)

사용자 데이터 유실된 경우(태그,레퍼런스없음)

해결 과정

Chain of Responsibility 패턴을 적용하여 OpenAI → Claude 순차 호출하는 Failover 구조를 설계했습니다. Outbox Pattern 기반 보상 트랜잭션과 @Scheduled 재시도 로직(30초 간격)으로 최종 일관성을 보장했습니다.

관련 기술 블로그

문제 상황

태그 연관관계 조회 시 수만 건의 데이터 처리로 평균 1.3초(최대 2초)의 응답 시간이 소요되어, 전체 페이지 로딩 속도까지 저하시켰습니다. 빈번한 조회로 DB 부하도 증가했습니다.

til 연관관계 그래프뷰 기능

함께 사용되는 til 조회 및 태그 상세 정보 조회 기능

해결 과정

변경 주기 분석을 통해 Local Memory Cache와 Spring Scheduler를 도입했습니다. 5분 주기 캐시 갱신으로 DB 쿼리를 수만 번 → 수백 번으로 대폭 감소시키며 데이터베이스 부하를 줄였습니다.

💾 Local Memory Cache

⏰ Spring Scheduler

5분 주기 캐시 갱신으로 최신 데이터 보장
